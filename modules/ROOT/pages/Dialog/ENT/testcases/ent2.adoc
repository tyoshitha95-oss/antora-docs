= Test Case: ENT2 - Username Authentication

== Description
Validates the scenario where Enterprise employees use isolated User/Password schemas tied to Domain identifiers (like `user@dialog.lk`).

== Configuration Setup (Simulated via cURL/API)

* **Account Type**: `ENT2`
* **Trigger Mechanism**: Username contains `@`.
* **Requested Username**: `dialog@corporate.lk`
* **Password**: `dialog123`

== Expected Execution Logic
1. Request arrives. Username contains `@`.
2. Router parses out the string. Sets `User-Name = dialog`.
3. Sets Group-ID to `ENT2`.
4. Executes Database check for User `dialog`.
5. Strictly matches `pap` layer.
6. Applies QoS profiles (e.g., `4M_BRONZE_T1_POLICE`) attached in the Vendor specific data tables upon Accounting Start.
