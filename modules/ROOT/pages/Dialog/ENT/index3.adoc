== Getting Started

To begin:

. Configure AAA server for enterprise authentication.
. Provision enterprise users via CRM or API integration.
. Define authentication method (Framed Route or Router-Based).
. Configure routing parameters (VRF, Loopback, Framed-IP, Routes).
. Set QoS templates and sub-template policies.
. Configure barring status for user control.
. Enable accounting for session tracking and CDR generation.
. Monitor authentication requests from NAS devices (PGW/LNS).
. Validate integration with external systems (CRM, BI, Network devices).

== Key Features

* Supports two authentication mechanisms:
  ** Framed Route Based Authentication
  ** Router Based Authentication
* Integration with multiple NAS devices (Huawei PGW, Cisco LNS/ASR).
* Real-time provisioning through CRM using API calls.
* Flexible routing configuration using VRF, Loopback, and Framed routes.
* QoS and firewall rules enforced using templates and sub-templates.
* Supports dynamic and static IP assignment handling.
* Accounting support with Tunnel-Link Start and Stop requests.
* CDR/EDR generation for enterprise users with BI integration.
* Idempotent provisioning (update existing users without duplication).
* Barring and unbarring functionality for service control.
