= System Architecture

== Overview
The Mac-Auth implementation heavily alters the standard FreeRADIUS server paradigm. Rather than depending on EAP or standard CHAP/PAP modules, it establishes a bespoke logic-gate pipeline directly evaluating MAC addresses and database-held state properties. 

At a high level, the architecture links:
* **NAS (Broadband Network Gateway - BNG)**: Usually Huawei BRAS components.
* **FreeRADIUS Engine**: The policy processor executing Unlang scripting.
* **Database**: Maintaining dynamic user state like quotas, locations, CDR limits, and active sessions (`MACL2USERS`, `L2RADONLINE`, `MACL2CDR`, `LOCTOPACKAGE`).

== Standard Data Flow
. *BNG* sends an `Access-Request` containing the user's MAC Address.
. **FreeRADIUS `authorize` block**:
  .. Checks if the MAC signature is formally legitimate.
  .. Validates the physical location.
  .. Identifies applicable `TIMESPAN` constraints dynamically.
  .. Resolves whether the packet is an *Initial Connect* or a *Continuation* (Service or FUP).
  .. Inspects data quotas to potentially step down the service.
  .. Asserts `Access-Accept` or `Access-Reject`.
. **FreeRADIUS `post-auth` block**:
  .. Commits dynamic decrements to user quotas (e.g., deducting cumulative Session Time and Remanent Volume).
  .. Updates concurrent state caching arrays.
. **FreeRADIUS `accounting` block**:
  .. Intersects `Stop` packets to finalize session billing and completely clear session data.
  .. Processes `Alive` interim packets.

== Core Components

=== Database Tools
Data relies entirely on SQL integrations defined in configuration wrappers (such as `sql_mac_auth1`, `sql_mac_auth2`, etc). The specific queries are externalized to ensure atomic read/write locks, preventing concurrent service queues from destructively overlapping quota numbers.

* **MACL2USERS**: Maps the user's MAC and Location ID to their structural package and active quotas.
* **L2RADONLINE**: Tracks all live connections per `ACCTSESSIONID` to monitor cumulative delta steps.

=== Variables Tree Tracking
The architecture deliberately eschews hard-coding properties into standard variables. It safely shifts data attributes into logical arrays (e.g., `Tmp-String-6`, `Control-UserExists`). 

`update control` blocks act as the lifeblood of this architecture, moving values internally without exposing logic strings arbitrarily back to the BNG.
