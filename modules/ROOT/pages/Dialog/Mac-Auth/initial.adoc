= Initial Authentication Use Cases

This page describes common scenarios handled during the Initial Authentication phase in MAC-based access control systems.

== 1. First Login (Unknown MAC)

When a device connects for the first time, its MAC address is not found in the system.

*Action:*
- Access is granted
- User/device is registered
- Quotas are initialized

*Example:*
A user connects to the network using a new smartphone. The system detects an unknown MAC address and assigns a fresh data quota.

== 2. Initial Login Within Reset Window

The device is recognized and the login occurs within the active quota period.

*Action:*
- Access is granted
- Existing quota is continued (no reset)

*Example:*
A user reconnects to the network later the same day. The remaining quota is preserved and reused.

== 3. Initial Login Beyond Reset Window

The device reconnects after the quota reset period has expired.

*Action:*
- Quotas are reset
- A new session is started

*Example:*
A user logs in the next day after exhausting the previous day’s quota. The system assigns a fresh quota.

== 4. Location Not Found

The system is unable to determine a valid location or access point for the device.

*Action:*
- Access is denied

*Example:*
A device attempts to connect through an unauthorized or unrecognized access point, resulting in rejection.

== 5. Access During Restricted Time

The login attempt occurs outside the allowed time window defined by policy.

*Action:*
- Access is denied

*Example:*
A user attempts to connect at midnight when access is restricted to daytime hours only.

== 6. Location Quota Exceeded (FUP)

The user exceeds the allocated quota under the Fair Usage Policy.

*Action:*
- Access is allowed
- Bandwidth is throttled

*Example:*
After consuming the full data quota, the user remains connected but experiences reduced network speed.
