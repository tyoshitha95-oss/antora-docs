= Scenarios 6–10 (Session Continuations)

== Overview
Continuation scenarios define how the system processes *active sessions* after initial authentication. These flows handle real-time quota tracking, delta calculations, and mid-session enforcement.

---

== Scenario 6: First Continuation Baseline

* **Trigger**:
** Service authentication (`Huawei-Service-Info = N...`)
** `ControlQT = 0`, `ControlQV = 0`
** `OldQT = 0`, `OldQV = 0`

* **Process**: Identified as initial continuation handshake
* **Outcome**:
** Treated as baseline (no delta deduction)
** Establishes tracking references for future calculations

== Scenario 7: Standard Delta Continuation

* **Trigger**: `ControlQT > OldQT`
* **Process**:
** Delta calculated → `QTDiff = ControlQT - OldQT`
** Deduction applied to `TimeQuota`
** `OldQT` updated to current value

* **Outcome**:
** Accurate quota tracking
** Seamless session continuation

== Scenario 8: Mid-Session Time Exhaustion

* **Trigger**: Delta reduces `TimeQuota → 0`
* **Process**: Flags `continuation_reject`
* **Outcome**:
** Immediate session termination
** BNG instructed to drop session

=== Request

NAS-Port=2138432,
NAS-IP-Address=10.174.157.1,
Service-Type=Framed-User,
Framed-Protocol=PPP,
Framed-IP-Address=10.254.157.235,
NAS-Identifier="MALA-B-HW-BNG-01",

NAS-Port-Id="\001QWLAN:wlan1:320:DE_IWAG_MAC:R310:997339611_DETAC_Ruckus_MAIN_AP:34:FA:9F:08:72:40:",

== Scenario 9: Mid-Session Volume Exhaustion

* **Trigger**:
** `VolumeQuota → 0`
** `TimeQuota > 0`

* **Process**:
** FUP transition is NOT allowed mid-session
** Flags `continuation_reject`

* **Outcome**:
** Session forcibly terminated
** User must reconnect to enter FUP flow (Scenario 4)

=== Request

[source,pseudocode]
----
echo 'User-Name="5a8c.7816.5f24",Acct-Session-Id="CONT004",ControlQV=314572800,OldQV=300000000' \
| radclient -x localhost auth mysecret
----

=== Expected Response

[source,pseudocode]
----
Access-Reject
    Reply-Message = "Volume quota exhausted. Reconnect required."
----

=== Database State

[source,sql]
----
SELECT USERNAME, VolumeQuota, TimeQuota FROM MACL2USERS;
----

|===
| USERNAME | VolumeQuota | TimeQuota

| 5a8c.7816.5f24
| 0
| 3600
|===

== Scenario 10: FUP Continuation

* **Trigger**: `Huawei-Service-Info = ..._H`
* **Process**:
** FUP mode enforced continuously
** Payload capped at `1048576 KB` (1GB logical window)
** No quota deduction applied to DB

* **Outcome**:
** Infinite volume access (rate-limited)
** Controlled by time constraints only

=== Request

[source,pseudocode]
----
echo 'User-Name="5a8c.7816.5f24",
NAS-Port=2138432,
NAS-IP-Address=10.174.157.1,
Service-Type=Framed-User,
Framed-Protocol=PPP,
Framed-IP-Address=10.254.157.235,
NAS-Identifier="MALA-B-HW-BNG-01",
NAS-Port-Type=Ethernet,
NAS-Port-Id="\001QWLAN:wlan1:320:DE_IWAG_MAC:R310:997339611_DETAC_Ruckus_MAIN_AP:34:FA:9F:08:72:40:",
Acct-Session-Id="MALA-B-00210SSG000100446996AAAbbc",
Huawei-Service-Info="NPostpaid_12M_UD_VOL"' \
| radclient -x localhost:21812 auth mysecret
----

=== Expected Response

[source,pseudocode]
----
Access-Reject:
Message-Authenticator = 0x27c7d3cd5c055ca90e415a336bbccaa3,
Reply-Message = "Volume quota exhausted. Please wait for quota reset interval."

Expected Access-Accept got Access-Reject

Reason:
IF used_volume >= allowed_volume THEN
    Reject access
    Reason = "Volume quota exhausted"
END IF
----
