= Scenarios 1–5 (Initial & FUP)

== Overview
These scenarios describe the complete connection lifecycle of a MAC-based user connecting through the BNG, covering initial onboarding, quota reset, normal usage, FUP enforcement, and rejection.

---

== Scenario 1: Brand New User

* **Trigger**: MAC not found (`MACL2USERS` returns 0 rows)
* **Process**: `mac_auth_handle_initial_login` → `initial_new`
* **Outcome**:
** New user inserted via `sql_mac_auth3`
** Quotas assigned from `PkgId`
** Access granted with default timeout

== Scenario 2: Returning User (Reset Window Passed)

* **Trigger**: `NOW() - LASTRSTTIME >= QTARSTINTERVAL`
* **Process**: `initial_reset`
* **Outcome**:
** Quotas reset to package maximum
** `LASTRSTTIME` updated
** User granted full access again

=== Request

[source,pseudocode]

echo 'User-Name="5a8c.7816.5f24",

== Scenario 3: Returning User (Mid-Window, Good Quota)

* **Trigger**: Within reset interval AND quotas available
* **Process**: `initial_update`
* **Outcome**:
** `LASTLOGINTIME` updated
** No quota reset
** Normal access granted

=== Request

[source,pseudocode]
----
echo 'User-Name="5a8c.7816.5f24",Acct-Session-Id="MID001"' \
| radclient -x localhost auth mysecret
----

=== Expected Response

[source,pseudocode]
----

    Session-Timeout = 7200
    Idle-Timeout    = 1800
----

== Scenario 5: Quota Exhaustion (Access Denied)

* **Trigger**: `TimeQuota = 0`
* **Process**: `initial_reject`
* **Outcome**:
** Access denied
** Informational message returned
** No DB modification
