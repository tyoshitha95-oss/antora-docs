= Performance Considerations

The GPON User Management API is designed to support high-volume provisioning, but operational performance depends on several parameters and usage patterns.

**Session and Idle Management**

* `maxusersessions` controls the maximum number of concurrent sessions per subscriber.  
  High values increase load on RADIUS tables; low values may disconnect active sessions.  
* `sessionlimit` defines maximum session duration in seconds. Recommended default is 48 hours (172800s).  
* `customnumfield1` defines idle timeout for automatic disconnection of inactive sessions. Recommended default is 24 hours (86400s).  
* Proper tuning of these values ensures efficient resource utilization and avoids RADIUS table exhaustion.

**Concurrency and Throughput**

* The API can handle batch provisioning of multiple subscribers.  
* High-frequency requests (e.g., large-scale migrations) may affect internal RADIUS processing.  
* Implement client-side throttling for large volumes to avoid backend performance degradation.  
* Repeated API calls with identical payloads are safe due to idempotency, but monitor `last_update` to detect actual changes.

**IP Assignment Impacts**

* Public Static IP: Larger payloads due to both IPv4 and IPv6 attributes.  
* Public Dynamic IP: Small payloads, but pool management may introduce internal delays if pool is exhausted.  
* Private Dynamic IP: Minimal payload, fastest processing.  
* Monitor pool usage to prevent allocation conflicts in high-volume systems.

**Recommendations**

* Use separate environments for testing high-volume provisioning.  
* Log API performance metrics (latency, response time, error rate).  
* Ensure session and idle limits are consistent across API calls to prevent unexpected disconnections.  
* Monitor backend RADIUS and OLT/ONT logs for delayed responses or failures.

**Additional Notes**

* Real-time monitoring of `barringstatus` and session parameters is recommended.  
* Plan IP pool allocations carefully for public dynamic assignments.  
* Optimize scripts to batch requests for subscribers sharing similar packages to reduce API load.  
* For compliance, record timestamp and user making the change in all automated operations.
